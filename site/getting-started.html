Guide

Getting started

Odyssey UI 无需构建步骤或包管理器。引入两个文件即可获得完整的组件样式与交互 —— 全部自包含、零外链。

安装

在页面 <head> 内引入样式表。odyssey.css 为核心（不内嵌字体，保持轻量）；odyssey-font.css 为可选的 Inter 字体（data-URI 内嵌，零外部请求）。

<head>
  <!-- 核心样式，必需 -->
  <link rel="stylesheet" href="https://odyssey.w33d.xyz/1.1/odyssey.css">

  <!-- Inter 字体，可选 -->
  <link rel="stylesheet" href="https://odyssey.w33d.xyz/1.1/odyssey-font.css">
</head>

在 </body> 前引入交互脚本。它零依赖、以 data-* 属性渐进增强，并在 DOMContentLoaded 自动初始化。若你只用静态组件，脚本可以省略。

  <script src="https://odyssey.w33d.xyz/1.1/odyssey.js"></script>
</body>

基本用法

组件类均以 .ody- 前缀命名，遵循 BEM（block .ody-x / element .ody-x__y / modifier .ody-x--z / 状态 .is-*）。直接写 class 即可，不会污染宿主页面的裸元素。

Ready 
<button class="ody-btn ody-btn--primary">Primary</button>
<button class="ody-btn ody-btn--secondary">Secondary</button>
<span class="ody-tag ody-tag--ok">
  <span class="ody-dot ody-dot--ok"></span> Ready
</span>

文档级排版（可选）

给任意容器加上 .ody-scope，即可让其内部的裸元素继承 Odyssey 的字体、颜色与盒模型基线 —— 适合整页文档。不加则库只作用于 .ody- 元素，绝不劫持宿主的 body／button／input。

交互（data-API）

脚本通过事件委托驱动，动态插入的标记无需重新初始化。常用属性：

  • data-ody-theme-toggle — 切换亮/暗主题并记忆到 localStorage。
  • data-ody-open="#id" — 打开 .ody-modal / .ody-drawer；data-ody-dismiss 关闭。
  • data-ody-tabs + data-ody-panel="#id" — 标签页。
  • data-ody-accordion + data-ody-collapse="#id" — 手风琴。
  • data-ody-copy="#sel" / data-ody-copy-text="字面" — 复制到剪贴板。
  • data-ody-tooltip="文本"、data-ody-popover="#id" — 提示与气泡卡片。

如需为后插入的子树手动接线，调用 Odyssey.init(rootEl)；命令式 Toast 用 Odyssey.toast({title, message, tone})。

动态 SSR 应用

公开的 .ody-* 发行包负责视觉与组件行为，不接管应用数据。同仓的 HOLDFAST 内部 Rust 发行线共享 Odyssey 设计语言，并提供 Wire、Spark 与 Motion：服务端继续生成可信 HTML，浏览器按 selector 局部交换，并为小型本地状态和列表变化增加渐进增强。

发行边界：/1.1/odyssey.js 只包含公开的 data-ody-* 组件行为，不包含 Wire、Spark 或 Motion，也不会发起网络请求。下面的 data-wire-* 示例仅适用于 vendored Rust 内部发行线。
Wire

HTML over the wire

同源 GET/POST、CSRF token 传递、History、焦点恢复与多个 target；响应 selector 不匹配时转为完整导航，网络或 HTTP 错误派发 wire:error 并显示 Toast。

Spark

Local state islands

用受限 data-API 管理显示、class、文本、属性与本地偏好，不引入通用状态框架。

Motion

Native continuity

基于 View Transition 与 FLIP，在 reduced-motion 下自动退回无动画路径。

<a href="?scope=wireguard"
   data-wire
   data-wire-target="#estate-results"
   data-wire-push>WireGuard</a>

<section id="estate-results" data-motion-list>...SSR HTML...</section>

<div data-spark="view:table" data-spark-persist="view=estate:view">
  <button data-spark-click="set:view=map">Map</button>
</div>

这条路径不是客户端 SPA，也不包含缓存、WebSocket 或离线数据库；服务端仍是唯一状态源。先在真实 vertical slice 中证明需求，再按需要增加传输能力。

主题切换

默认亮色。暗色同时由两条途径触发，显式覆盖优先：

  • 系统偏好 — @media (prefers-color-scheme: dark) 自动生效。
  • 显式属性 — 在 <html>（或任意祖先）设 data-theme="dark" / "light"。

给任意按钮加 data-ody-theme-toggle，脚本便会在 <html> 上切换 data-theme 并写入 localStorage('ody-theme')，下次自动读回。右上角的图标按钮正是这样接线的 —— 点它试试。

<!-- 声明式：任意元素加属性即可 -->
<button class="ody-btn ody-btn--secondary" data-ody-theme-toggle>切换主题</button>

<!-- 命令式 -->
<script>
  Odyssey.setTheme('dark');   // 或 'light'
  Odyssey.toggleTheme();
</script>

CDN 与自托管

两种方式二选一，产物文件完全相同且自包含：

  1. CDN — 直接指向发行地址 https://odyssey.w33d.xyz/1.1/…，最省事。
  2. 自托管 — 把 dist/ 下的 odyssey.css、odyssey-font.css、odyssey.js 拷进你的静态资源目录，改成本地路径。因为无任何外链，离线／内网／气隙环境同样可用。

2引入 JS
